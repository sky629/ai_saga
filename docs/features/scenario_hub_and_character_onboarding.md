## 목표

- 메인 진입 화면을 `내 스토리` 중심에서 `시나리오 탐색` 중심으로 전환한다.
- 기존 세션 목록은 별도 `내 스토리` 화면으로 분리한다.
- 캐릭터 생성 흐름을 이름 자유 입력 + 선택지형 온보딩으로 확장한다.

## 범위

### In Scope

- 메인 화면에서 시나리오 카드 목록 노출
- 시나리오 선택 후 캐릭터 온보딩 모달/플로우
- 캐릭터 프로필 구조 추가 및 저장
- 캐릭터 프로필을 초기/턴 프롬프트에 반영
- `내 스토리` 화면 신설 및 기존 세션 목록 이동

### Out of Scope

- 시나리오 상세 페이지
- 시나리오 태그/대표 이미지 영속화
- 공개 세션 관람 기능
- 로어북 시스템 도입
- 캐릭터 온보딩 기반의 스탯/클래스 분기

## 수용 기준

1. `/` 접속 시 사용자는 시나리오 목록을 본다.
2. 사용자는 시나리오 카드에서 바로 캐릭터 생성 플로우로 진입할 수 있다.
3. 캐릭터 생성 시 다음 정보를 입력할 수 있다.
   - 이름
   - 나이
   - 성별
   - 외형
   - 특징 1~2개 (선택)
   - 목표 (선택)
4. 생성된 캐릭터 프로필은 저장되고, API 응답에서 조회 가능하다.
5. 게임 시작/액션 처리 시 캐릭터 프로필 요약이 프롬프트에 반영된다.
6. 기존 세션 목록은 `/stories`에서 계속 접근 가능하다.
7. 게임 화면의 뒤로 가기는 `내 스토리`로 연결된다.

## API / 데이터 모델 영향

### Character

- `CharacterEntity`에 `profile` 구조 추가
- `profile`은 확장 가능한 JSON 구조로 저장
- 기존 `description`은 제거하고 프롬프트/표시는 `profile` 기반으로만 처리

### Create Character Request

- 기존
  - `name`
  - `scenario_id`
- 변경
  - `name`
  - `scenario_id`
  - `profile.age`
  - `profile.gender`
  - `profile.appearance`
  - `profile.traits[]`
  - `profile.goal`

### Character Response

- 기존 필드 유지
- `profile` 추가

## 흐름

1. 메인 화면이 시나리오 목록을 조회한다.
2. 사용자가 시나리오를 선택한다.
3. 설문형 캐릭터 온보딩 모달이 열린다.
4. 프론트는 온보딩 데이터를 포함해 캐릭터 생성 API를 호출한다.
5. 백엔드는 캐릭터 프로필을 저장한다.
6. 프론트는 생성된 캐릭터로 세션 시작 API를 호출한다.
7. 시작/액션 프롬프트는 캐릭터 프로필 요약 문자열을 사용한다.

## 엣지 케이스 / 실패 모드

- 특징과 목표는 비어 있어도 캐릭터 생성은 가능해야 한다.
- 나이, 성별, 외형은 빈 값이면 검증 실패해야 한다.
- 특징은 최대 2개까지만 허용해야 한다.
- 시나리오가 비어 있으면 메인 화면에서 재시도 UI를 보여준다.

## 구현 단계

1. 백엔드 계약/엔티티/매퍼/응답 확장
2. 백엔드 단위 테스트 및 회귀 테스트 추가
3. 프론트 라우팅 개편 (`/` 시나리오 허브, `/stories` 내 스토리)
4. 프론트 설문형 온보딩 모달 구현
5. 프론트 API 타입/호출 갱신
6. 빌드/테스트/QA 체크

## 테스트 계획

### Backend

- 캐릭터 프로필 포함 생성 유스케이스 단위 테스트
- 요청/응답 스키마 회귀 테스트
- 캐릭터 저장/조회 통합 테스트
- 시작 프롬프트에 프로필 요약 반영 테스트

### Frontend

- 타입 체크/빌드 통과
- 시나리오 허브 렌더링 확인
- 온보딩 입력 → 캐릭터 생성 → 세션 시작 흐름 확인
- `내 스토리` 세션 목록/삭제/이동 흐름 회귀 확인
